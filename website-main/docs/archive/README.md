# Archive

One-time session reports and audits — a record of "here's what changed in
this pass," not documentation anyone needs to read going forward. Nothing in
this folder is referenced by any code, build script, or config; it's kept
purely for historical reference. Safe to delete entirely if you don't want
the history.

If you're looking for current, still-relevant documentation, see
[`../need-to-know/`](../need-to-know/README.md) instead.

## Structure

- **`admin-reports/`** — write-ups from building/refining the Studio-Ops admin dashboard (navigation, theming, security review, UI refinement passes)
- **`seo-reports/`** — SEO strategy docs and audits (Google indexing, entity strategy, technical SEO passes)
- **`security-reports/`** — point-in-time security audits (env/runtime audit, hardening pass, security reviews)
- **`production-reports/`** — "final polish" / launch-readiness reports from various production push passes
- **`business-reports/`** — business-logic build reports (billing foundation, finance/receipts system, positioning, cache fixes)
