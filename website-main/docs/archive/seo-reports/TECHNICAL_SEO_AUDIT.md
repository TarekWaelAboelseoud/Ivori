Date: 2026-05-22

## Metadata

- Centralized defaults in `lib/seo/site.ts`.

## Crawl And Indexing

- Sitemap includes home, Arabic home, public service routes, work routes, and note routes.
- KVL flagship work has increased sitemap priority.
- Notes are crawlable and now have Article schema.
- Robots remains clean and does not advertise admin/API/operator paths.

## Performance Notes

- No redesign or heavy client-side features were added.
- Structured data is server-rendered.
- Icons are generated by App Router metadata routes and statically optimized.
- Existing `next/font` preload behavior remains intact.
- Admin hardening remains dynamic/no-store.

## Remaining Technical Risks

- The CSP still allows `unsafe-inline` for scripts/styles from the previous security pass to preserve hydration and current styling. A nonce/hash CSP would be the next stricter security step but is more invasive.
- Contact is a client page and does not currently export dedicated route metadata. It still inherits global metadata; a future server wrapper can add exact contact metadata without changing UX.
- Route-specific OG images are still global. Dedicated OG images for KVL and service pages would improve share specificity.
